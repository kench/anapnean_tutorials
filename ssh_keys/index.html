Using SSH keys (*NIX)
This guide assumes that you're already able to connect to Anapnea via SSH using a password. If not, please read the connection guide for Linux or Mac OS X.

SSH public key authentication allows you to use one identity to connect to multiple servers. It also provides enhanced security by ensuring that you password is never transmitted.
Open a Terminal and type ssh-keygen -t rsa. This will generate a public/private key pair used for authentication.
You can leave the key file name to its default.

Use a passphrase! Anyone that has access to your private key will be able to log in as you!

Note: Keep your private key secure! Don't give it out to others or leave it in a insecure location.
Copy the SSH public key to the proper location on the remote server. Replace username with your Anapnea username.

scp .ssh/id_rsa.pub username@anapnea.net:.ssh/authorized_keys

Note: This will overwrite any existing authorized keys for your remote account.
Try connecting to Anapnea via SSH. You should not have to enter your password to log in, unless you have used a passphrase with your key, which is not sent to the server.
